# Spec：灵感知识库

## 一、What

一个跨平台的灵感捕获与知识沉淀工具。

用户从任何来源快速存入灵感——URL、文字、截图、视频片段、音频片段、任意文件——系统自动完成结构化消化，所有数据以开放文件格式本地存储，在统一画布上浏览和整理。

## 二、Why

创作者在日常中不断接触大量灵感来源，但现有工具的收藏/书签功能存在三个根本缺陷：

1. **平台绑定** — 内容锁在各个 App 的收藏夹里，链接失效即丢失，数据不归用户所有
2. **无消化** — 存了就存了，没有自动梳理和关联，积累越多越混乱
3. **不可持续** — 素材散落在各处（便签、txt、文件夹、收藏夹），彼此孤立，无法形成可生长的知识体系

## 三、How（核心机制）

**捕获：** 多入口统一接收——PC 端快捷浮窗、移动端系统分享菜单。输入内容不限于 URL 和文本，也支持图片、视频、音频等全格式附件直接存入。

**消化：** AI 固定管线自动处理每条灵感——分类、提取关键信息、打标签、识别归属主题、生成摘要、发现与已有灵感的关联。消化能力按媒体格式渐进覆盖：文本和图片当前已成熟，视频和音频随模型能力提升逐步接入。

**浏览：** 统一无限画布承载所有灵感。卡片以节点形式存在，关联以连线呈现，整体构成可交互的关联图谱。用户可随意拖拽排列、手动连线、分组、修正 AI 的分类结果。

**数据：** 灵感以开放文件格式（JSON + Markdown + 附件）存储在本地。数据结构天然可被外部 agent 框架（如 Codex）直接读写，深度分析在外部环境中完成。

## 四、Effect（用户价值链路）

```
    碎片灵感                   结构化知识                   创意产出
    ──────────────────────────────────────────────────────────→

    捕获 → 消化 → 浏览 ──→ 深度反刍（外部 agent）
    （系统）              （用户 + AI 协作）
```

系统负责把无序的碎片转化为有序、可检索、相互关联的结构化数据。用户在此之上做深度分析，把模糊直觉推向可落地方案。

## 五、边界

**明确属于本产品：**
- 多渠道灵感捕获与存储
- AI 自动分类、标签、摘要、关联
- 画布可视化浏览与手动整理
- 开放文件格式的数据沉淀
- 全格式附件存储（文本/图片/视频/音频/文件）

**明确不属于本产品：**
- 深度反刍与 agent 对话——由外部 agent 框架完成
- 内容抓取与反爬——URL 内容获取 best-effort，不做对抗
- 设备间同步——依赖外部工具（Syncthing/iCloud）
- 社交分享与协作——纯个人工具
- 自动化采集——不 RSS、不监控、不定时任务
- 内容发布——不发博客、不导出文章

**已知约束：**
- 封闭平台（微信、微博、贴吧等）内容需用户手动截图，无法自动获取
- 视频消化的 AI 能力局限于视觉帧，音频轨暂不支持
- 音频附件的 AI 消化暂无能力——消化质量依赖用户简评
- 大体积附件会增加同步流量和存储开销
- 移动端画布操作受限于触屏交互，复杂编排以 PC 为主

## 六、验收标准

产品完成时，以下应为真：

- 用户可在 PC 和手机上捕获灵感（URL / 文字 / 图片 / 文件），提交后无需等待
- AI 自动完成分类、标签、摘要，并发现与已有灵感的语义关联
- 所有灵感以开放文件格式存储在本地指定目录
- 用户可在画布上浏览所有卡片，节点按主题差异化呈现
- 用户可拖拽排列节点、手动连线、修改 AI 分类结果
- 搜索可按关键词、主题、类型、时间范围找到目标灵感
- 数据目录可通过文件同步工具在设备间保持一致
- 无网络时浏览、编辑、搜索均正常工作
